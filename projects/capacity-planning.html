Project Overview

The Capacity Planning Tool (CPT) manages workforce distribution across hundreds of branches. It solves the critical challenge of maintaining a \"Master Blueprint\" of recurring shifts while allowing for \"Live Schedule\" modifications (sick leaves, swaps) without data corruption.

Key Features

  • Dual-Layer Scheduling: Separation of Master Templates and Live Instances.
  • Automated Conflict Detection: Real-time validation of vacation overlaps and shift illegalities.
  • Temporal Data Snapshots: Historical tracking for payroll and audit accuracy.
  • Multi-Country RBAC: Granular permissions based on regional hierarchy.
  • Biweekly Parity Engine: Complex logic for rotating even/odd week shift patterns.

Frontend Architecture

Built as a static SPA for maximum performance and security.

  • Provider Stack: AuthProvider > QueryClientProvider > UIProvider.
  • State Management: Server-state via TanStack Query; UI-state via React Context.
  • Token Storage: Secure HTTP-Only cookies with client-side memory fallback.

Backend Architecture

Django-based REST API focusing on domain-driven logic.

  • Middleware: Custom Auth Middleware for enterprise SSO integration.
  • Validation: DRF Serializers with deep business logic injection.
  • Replication: Automated daily sync tasks for historical archiving.

Two-Table Shift Engine

This core logic ensures that changes to a specific day's shift do not affect the recurring template, and vice versa.

Table Purpose Update Frequency
master_shifts The \"Blueprint\" - defines recurring weekly patterns. Rare (Contractual changes)
live_shifts The \"Instance\" - handles daily reality and disruptions. High (Daily operations)

Replication Flow: A background worker generates live_shifts from master_shifts 180 days in advance. Any manual edit to a live_shift flags it as \"detached\" to prevent override during blueprint updates.
